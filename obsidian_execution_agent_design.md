# Obsidian Execution Agent 설계안

## 1. Agent 개요

### 목표
Obsidian vault 안의 노트, 할 일, 일정, 아이디어를 읽어서  
단순 Q&A를 넘어서 **질문 응답, 계획 수립, 우선순위 판단, 병목 진단, 집중 세션 추천, overdue task 재조정, 노트 생성/수정**까지 수행하는 개인 실행 지원 agent를 만든다.

### 핵심 방향
이 agent는 단순한 챗봇이 아니라 아래를 수행하는 **운영형 personal agent**를 지향한다.

- vault 내용을 읽고 현재 상태를 파악
- 사용자의 목표와 아이디어를 실행 가능한 계획으로 변환
- 일정과 제약을 고려하여 현실적인 조정안을 제시
- 필요한 경우 markdown 노트를 직접 생성하거나 수정
- 반복적으로 미뤄지는 할 일을 다시 굴러가게 만드는 실행 지원

---

## 2. 왜 이 agent가 필요한가

사용자의 vault는 단순 메모 저장소가 아니라 다음이 함께 들어 있는 운영 시스템에 가깝다.

- 연구 및 논문 준비
- 회사 업무 및 기술 학습
- 스터디 및 개인 프로젝트
- 일상적인 할 일 관리

이런 환경에서는 단순 검색형 Q&A만으로는 부족하다.  
필요한 것은 다음이다.

1. 지금 내가 뭘 하고 있는지 파악
2. 어떤 일이 중요한지 판단
3. 왜 진행이 막히는지 진단
4. 오늘 무엇을 해야 하는지 추천
5. 미뤄진 일을 다시 일정에 반영
6. 결과를 노트에 남겨 다음 실행으로 연결

즉, 이 agent의 핵심은 **정리**가 아니라 **실행 지원**이다.

---

## 3. Agent의 전체 역할

이 agent는 아래 3개 축을 동시에 가진다.

### A. Knowledge Assistant
- vault 기반 Q&A
- 특정 주제 관련 맥락 요약
- 관련 노트 연결

### B. Planning Assistant
- 목표를 WBS로 분해
- 마감일 기반 일정안 생성
- 우선순위 추천
- 일정 충돌과 현실 제약 반영

### C. Execution Assistant
- 병목 진단
- 오늘의 집중 task 제안
- overdue task 재조정
- markdown 노트 생성 및 수정

---

## 4. 주요 모드 설계

## 4.1 Q&A Mode

### 목적
vault 내용을 바탕으로 질문에 답하고, 관련 노트들을 연결해서 맥락을 보여준다.

### 예시 입력
- 내 논문 방향 뭐였지?
- BERT4Rec 관련해서 내가 정리한 내용 보여줘
- 내가 toy agent 아이디어로 뭘 생각하고 있었지?

### 주요 동작
1. 질문에서 핵심 키워드 추출
2. 관련 노트 검색
3. 상위 관련 노트 요약
4. 답변 생성
5. 관련 노트 목록 제시

### 출력 예시
- 현재 논문 방향은 BERT4Rec 기반 행동 예측과 시간 정보 통합이다.
- 관련 노트:
  - 논문.md
  - 논문 - 주제 상세.md
  - 교수님 미팅.md

---

## 4.2 Planning Mode

### 목적
사용자가 가진 목표나 아이디어를 일정과 제약을 고려한 실행 계획으로 바꾼다.

### 예시 입력
- 논문 초안을 5월 말까지 쓰고 싶어
- codex toy agent를 2주 안에 하나 만들고 싶어
- pytest 공부를 이번 달 안에 끝내고 싶어

### 주요 동작
1. 목표 분석
2. 목표를 작업 단위로 분해
3. WBS 생성
4. 완료 목표일 기준 역산
5. 사용자의 고정 일정 및 현실 제약 반영
6. 리스크 확인
7. 조정안 제시

### 출력 예시
- WBS
- 주차별 계획
- 리스크
- 추천 전략

---

## 4.3 Action Mode

### 목적
분석이나 계획 결과를 실제 Obsidian markdown 노트로 생성하거나 기존 노트를 수정한다.

### 예시 입력
- 논문 실행 계획 노트 만들어줘
- 이번 주 우선순위 노트로 정리해줘
- overdue task 재조정 결과를 할 일 노트에 반영해줘

### 주요 동작
1. 생성/수정 대상 노트 결정
2. markdown 템플릿 생성
3. 필요한 내용 채워넣기
4. 파일 생성 또는 수정

### 출력 예시
- `논문_실행계획.md` 생성
- `이번주_우선순위.md` 갱신
- `할 일.md` 내 overdue task 날짜 수정

---

## 4.4 Priority Optimization Mode

### 목적
현재 vault 안의 여러 일 중에서 지금 가장 먼저 해야 할 일을 뽑아준다.

### 예시 입력
- 이번 주 뭐 해야 해?
- 지금 제일 중요한 일 3개만 골라줘
- 논문, 업무, 스터디 중 뭐부터 해야 할까?

### 주요 동작
1. 프로젝트/할 일/최근 수정 노트 수집
2. 중요도 판단
3. 긴급도 판단
4. 선행 조건 여부 확인
5. 시간 대비 효과 고려
6. 우선순위 목록 생성

### 출력 예시
1. 데이터셋 확정
2. baseline 구현
3. pytest 테스트 케이스 정리

---

## 4.5 Bottleneck Detector Mode

### 목적
왜 일이 밀리거나 진행이 안 되는지 병목을 진단한다.

### 예시 입력
- 논문 진행이 왜 안 되고 있는지 봐줘
- 지금 내가 막혀 있는 지점이 뭐야?
- 프로젝트 진행 병목 분석해줘

### 주요 동작
1. 관련 노트 및 task 수집
2. 반복 등장하는 미완료 항목 파악
3. dependency 확인
4. 선행 조건 미완료 여부 탐지
5. 병목 원인 추론

### 출력 예시
- dataset 미확정으로 실험이 시작되지 못함
- baseline 구현이 늦어져 비교 실험 일정이 밀리고 있음
- 관련 자료는 있으나 결정 노트가 없음

---

## 4.6 Focus Session Mode

### 목적
지금 확보된 시간 안에서 당장 집중할 수 있는 작업을 제안한다.

### 예시 입력
- 오늘 2시간 뭐할까?
- 지금 1시간밖에 없는데 뭘 하면 좋을까?
- 이번 저녁에 할 수 있는 가장 적절한 작업 추천해줘

### 주요 동작
1. 현재 우선순위 확인
2. overdue task 여부 확인
3. 병목 관련 task 우선 고려
4. available time에 맞는 작업 후보 추출
5. 구체적 집중 범위 정의

### 출력 예시
- 오늘의 focus task: CASAS vs BPI 비교 정리
- 권장 시간: 90분
- 목표: 데이터셋 하나로 좁히기
- 완료 기준: 비교 표 완성

---

## 4.7 Overdue Task Rescheduler Mode

### 목적
미뤄진 할 일을 찾아서 중요도와 제약을 고려해 현실적으로 다시 재배치한다.

### 예시 입력
- overdue task 정리해줘
- 미뤄진 일들 이번 주 기준으로 다시 배치해줘
- 날짜 지난 task들 새 일정으로 옮겨줘

### 주요 동작
1. due date가 있는 task 파싱
2. overdue task 식별
3. task의 프로젝트/주제/의존성 확인
4. 중요도 및 긴급도 재평가
5. 새 날짜 후보 계산
6. 충돌 여부 확인
7. 제안안 생성
8. 승인 시 markdown 수정

### 출력 예시
- 데이터셋 확정 → 4/5로 재배치
- baseline 구현 → 4/8로 재배치
- 블로그 초안 → 다음 주로 이동

---

## 5. 모드 간 연결 구조

이 agent의 강점은 각 모드가 따로 노는 것이 아니라 연결된다는 점이다.

### 흐름 예시 1
Q&A → Planning
- “내 논문 방향 뭐였지?”
- 관련 내용 확인 후
- “그럼 5월까지 끝내는 계획 짜줘”

### 흐름 예시 2
Planning → Bottleneck → Priority
- 목표 기반 WBS 생성
- 병목 분석
- 이번 주 우선순위 재정렬

### 흐름 예시 3
Priority → Focus → Action
- 우선순위 선정
- 오늘 집중 작업 선택
- focus note 생성

### 흐름 예시 4
Overdue → Priority → Action
- overdue task 탐지
- 중요도 기반 재조정
- 노트에 새 날짜 반영

---

## 6. 추천 동작 방식: Intent Routing

사용자가 매번 모드를 명시하지 않아도 agent가 입력을 보고 적절한 모드를 선택하도록 설계한다.

### 예시
- “내 논문 방향 뭐였지?” → Q&A Mode
- “논문 5월까지 끝낼 수 있을까?” → Planning Mode
- “지금 제일 중요한 일 뭐야?” → Priority Optimization Mode
- “왜 논문이 안 풀리는지 봐줘” → Bottleneck Detector Mode
- “오늘 2시간 뭐할까?” → Focus Session Mode
- “overdue task 다시 잡아줘” → Overdue Task Rescheduler Mode
- “정리 결과 노트로 만들어줘” → Action Mode

## 7. 입력 데이터 설계

이 agent가 활용하는 주요 입력은 아래와 같다.

### 7.1 Vault 노트
- 연구 노트
- 프로젝트 메모
- 회의 메모
- 아이디어 정리
- 공부 기록

### 7.2 Task 데이터
- checkbox task
- due date
- project 관련 tag
- 우선순위 표시가 있다면 반영

### 7.3 일정/제약 정보
- 회사 업무 시간
- 대학원 수업 일정
- 사용자가 따로 적어둔 고정 일정
- 특정 기간 내 마감일

### 7.4 메타데이터
- 최근 수정일
- 노트 경로
- 태그
- 파일명 패턴

---

## 8. 출력 문서 예시

이 agent는 대화 응답뿐 아니라 아래 같은 결과물을 만들어낼 수 있다.

- `논문_실행계획.md`
- `이번주_우선순위.md`
- `오늘의_focus.md`
- `병목_분석.md`
- `overdue_task_재조정안.md`
- `프로젝트_대시보드.md`

이렇게 되면 대화가 끝나도 결과가 남고, 다음 실행에 다시 활용할 수 있다.

## 11. 차별점

이 agent의 차별점은 아래와 같다.

### 1. 단순 Q&A가 아님
질문에 답하는 것에서 끝나지 않고 계획과 실행으로 이어진다.

### 2. 사용자의 실제 vault를 활용
일반적인 생산성 agent가 아니라 사용자의 실제 상황과 노트를 기반으로 동작한다.

### 3. 미뤄진 일을 다시 굴러가게 만듦
overdue task 재조정 기능은 체감 가치가 크다.

### 4. 결과를 실제 파일로 남김
대화형 assistant가 아니라 markdown 기반 작업 보조 시스템이 된다.

---

## 12. 최종 한 줄 정의

**Obsidian Execution Agent는 vault 안의 지식, 목표, 할 일, 일정을 연결하여  
질문 응답부터 계획 수립, 우선순위 판단, 병목 진단, 집중 추천, overdue 재조정, 노트 생성/수정까지 수행하는 personal execution agent이다.**
