# Obsidian Agent 테스트 계획

## 테스트 목적
agent가 특정 주제에 치우치지 않고 다양한 업무 문서를 읽고 답변할 수 있는지 확인한다.

## 테스트 문서 유형
- 프로젝트 일일 보고
- 프로젝트 리스크 로그
- 교육 수강 계획
- 교육 학습 기록 템플릿
- 프로젝트 회의록
- 팀 주간회의
- agent 개요 문서

## Q&A 테스트 후보
- 신규 팀원이 설정 전에 확인해야 할 것은 무엇인가?
- 배포나 릴리스 전에 어떤 항목을 확인해야 하는가?
- 교육 수강 계획의 이번 주 할 일은 무엇인가?

## Planning 테스트 후보
- 이번 주 안에 팀 온보딩 문서를 개선하고 싶다.
- 4주 동안 Python 테스트 자동화 교육을 운영하고 싶다.
- 설정 검증 기능을 하루 안에 추가하고 싶다.

## 성공 기준
- 관련 문서를 올바르게 찾는다.
- 답변에 관련 노트와 다음 액션이 포함된다.
- 계획 응답에는 WBS, 우선순위, 리스크가 포함된다.
