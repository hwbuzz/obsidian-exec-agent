# 프로젝트 리스크 로그

## 주요 리스크

### 설정 오류
- 사용자가 필수 환경 변수를 누락할 가능성이 높다.
- 설정 경로가 운영체제마다 다르게 동작할 수 있다.

### 테스트 데이터 부족
- 실제 업무와 비슷한 샘플 문서가 부족하면 기능 검증이 어렵다.
- 예시가 특정 주제에 치우치면 agent 성능을 일반화하기 어렵다.

### 문서화 지연
- 구현 후 사용법 문서가 늦어지면 팀원이 바로 테스트하기 어렵다.
- 변경 사항이 README와 맞지 않으면 혼란이 생긴다.

## 대응 방안
- 설정 확인 명령을 제공한다.
- 샘플 vault를 별도로 관리한다.
- 기능 추가 시 테스트 예시와 사용법을 함께 업데이트한다.
