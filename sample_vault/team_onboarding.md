# Team Onboarding Guide

## 목적
새 팀원이 첫 주에 개발 환경, 협업 규칙, 문서 위치를 빠르게 파악하도록 돕는다.

## 첫 주 체크리스트
- [ ] 저장소 접근 권한 확인
- [ ] 로컬 개발 환경 설치
- [ ] 팀 캘린더 구독
- [ ] 주요 문서 위치 확인
- [ ] 첫 번째 작은 이슈 배정

## 협업 규칙
- 작업 시작 전 이슈에 현재 상태를 남긴다.
- PR은 작게 만들고, 변경 이유를 설명한다.
- 회의 결정사항은 당일 문서에 기록한다.

## 자주 막히는 지점
- 권한 요청 담당자를 모르는 경우
- 로컬 환경 변수 이름이 문서와 다른 경우
- 리뷰 요청 시점을 늦게 잡는 경우
