# Release Checklist

## 목적
작은 기능 배포 전 확인해야 할 항목을 정리한다.

## 배포 전 확인
- [ ] 변경 범위가 릴리스 노트에 정리되어 있다.
- [ ] 기본 테스트가 통과했다.
- [ ] 주요 사용자 흐름을 수동으로 확인했다.
- [ ] 되돌리기 방법이 정리되어 있다.
- [ ] 배포 후 모니터링 담당자가 정해져 있다.

## 리스크
- 테스트 데이터와 운영 데이터의 차이를 놓칠 수 있다.
- 배포 후 확인할 지표가 없으면 문제 감지가 늦어진다.
- 롤백 절차가 모호하면 장애 대응 시간이 길어진다.

## 권장 배포 순서
1. 변경 범위 확인
2. 테스트 결과 확인
3. 릴리스 노트 작성
4. 배포
5. 배포 후 로그와 주요 지표 확인
